总样本 N = TP+TN+FP+FN

以投诉为例子的话，这些指标的含义如下

符号 英文 含义 以投诉为例子
TP True Positive 预测为正且实际为正 预测为投诉且真的投诉
FP False Positive 预测为正但实际为负 预测为投诉但实际没投诉（误杀）
FN False Negative 预测为负但实际为正 预测为不投诉但真实投诉（漏网）
TN True Negative 预测为负且实际为负 预测为不投诉且确实没投诉

引申指标介绍

指标 英文/同义词 公式 业务一句话解释 阈值↓
Threshold 决策阈值 用户自定义 概率≥该值则判为正例 —（自变量）
Accuracy 准确率 (TP+TN)/N 全体样本里判对的比例 先↑后↓（不平衡时骤降）
Error Rate 错误率 1−Accuracy 全体样本里判错的比例 先↓后↑
Precision 查准率 TP/(TP+FP) “预测为正”里真正正例的比例 ↓（FP↑更快）
Recall 召回率/TPR/Sensitivity TP/(TP+FN) 所有真实正例里找回的比例 ↑（FN↓）
F1 - 2·P·R/(P+R) 精确与召回的调和平均 先↑后降（存在峰值）
FPR 假正率 FP/(FP+TN) 所有真实负例里被误杀的比例 ↑（FP↑）
Specificity 特异度/真负率 TN/(TN+FP) 所有真实负例里被正确保护的比例 ↓（TN↓）
Prevalence 患病率/先验 (TP+FN)/N 总体里真实正例的占比 不变（与阈值无关）

ROC-PR

曲线 横轴 (X) 纵轴 (Y) 面积名称 基线值 对类别不平衡敏感? 业务一句话解释 实际用处
ROC FPR TPR AUC-ROC 0.5 (对角线) 不敏感 随机抽一正一负，正样本得分更高的概率 • 快速筛模型：AUC<0.7 基本不可用
• 平衡数据下跨模型对比
PR Recall Precision AUC-PR (=AP) π = Prevalence (正例占比) 极敏感 随机抽一个正例，它比随机负例排名靠前的平均精确度 • 不平衡场景首选：投诉/欺诈/疾病
• 同模型不同阈值选优
• 基线=π，远离π才真提升

补充指标

阈值下降：放松，抓更多正例；阈值上升：收紧，少报正例

指标 公式 阈值↓ 阈值↑ 业务一句话解释 实战用处
MCC (TP·TN - FP·FN) / √[(TP+FP)(TP+FN)(TN+FP)(TN+FN)] 先↑后↓（存在峰值） 先↑后↓（同峰值） 四格表全考虑的综合相关系数，+1 完美，0 随机，-1 完全反向 • 极度不平衡时代替 Accuracy 做模型初选
• 跨数据集比较更稳健
Fβ (1+β²)·(P·R) / (β²·P + R) 峰值位置随β左移（β>1 重 Recall） 峰值位置随β右移（β<1 重 Precision） 加权 F1：β>1 更怕漏，β<1 更怕误报 • 业务给代价即可反推β
• 选β=2 常用在“漏报损失大”场景（医疗、投诉）

阈值滑动时，可以考虑F1, MCC. Fβ 这三个值，这三个值的峰值对应如下解释：

  • F1 峰值 ≈ 精确率与召回率“折中”最优点；
  • MCC 峰值 ≈ 整体四格表“最协调”点；
  • Fβ 峰值 ≈ 按业务代价（β）加权后的“最划算”点。

对应到业务逻辑上：

  • 如果代价表明确实“漏报更贵”，就直接用 Fβ 峰值的阈值；
  • 若只想纯粹综合性能，可用 MCC 峰值；
  • 没有额外偏好，用 F1 峰值也不会错。

因此，对于大部分情况而言F1值就够应付任务。

概率校准指标

暂时没用到，但是有用，总之先记着

指标 公式 业务解释 判断好坏
Brier Score mean( (p − y)² ) “预测概率”与真实 0/1 标签的均方误差 0 完美，上限 1 越差
LogLoss −mean[ y·log p + (1−y)·log(1−p) ] 对数似然惩罚，极端概率（0.001/0.999）错一点就爆炸 0 完美，+∞ 最差

投诉工单数据分析

步骤：原始字段 → 数据清洗 → 探索性分析(EDA) → 特征构造 → 特征选择 → 特征变换 → 评估与优化

原始字段

理解业务含义，确定数据类型。需要创建数据字典，标注出字段角色，比如确定用户表中的最后登陆时间。

数据清洗

处理缺失、异常、重复等数据，但是不进行统计分析。

探索性数据分析（EDA）

  • 形状和字段类型：行数、列数、类型比例
  • 缺失：缺失率、缺失模式
  • 重复和异常：重复行占比、异常占比
  • 分布形态

进行可视化图表绘制，对特征进行初步筛选，只进行观察和分析。

探索性数据分析|Exploratory Data Analysis|EDA 入门（基本概念、流程、工具及资源）

note

特征工程（Feature Engineering）就是从原始数据中提取特征并将其转换为适合机器学习的格式，即将原始的“脏数据”通过清洗和整合，转化为适合机器学习模型的特征，从而改善机器学习性能，提高预测准确度，同时节省运行时间。

特征构造

剔除冗余、噪声、无关特征，降低维度和过拟合风险。

特征选择

特征选择是特征工程里的一个重要问题，其目标是寻找最优特征子集。特征选择能剔除不相关(irrelevant)或冗余(redundant )的特征，从而达到减少特征个数，提高模型精确度，减少运行时间的目的。
